# MCP Server for ReqAuth API

Expose ReqAuth API as MCP tools over stdio so an LLM can call the API safely.

Generated by **mcpforge** at 2026-02-27T18:00:13.945Z.

## Prerequisites

- Node.js 18+
- npm 9+

## Setup

1. Install dependencies:

```bash
npm install
```

2. Configure environment:

```bash
cp .env.example .env
```
The `ACCESS_TOKEN` value is required for authenticated endpoints.

3. Build:

```bash
npm run build
```

4. Run:

```bash
npm start
```

## Claude Desktop Configuration

Add this to your Claude Desktop MCP config file (adjust the absolute path):

```json
{
  "mcpServers": {
    ".tmp-reqauth-server": {
      "command": "node",
      "args": ["/ABSOLUTE/PATH/TO/.tmp-reqauth-server/dist/index.js"],
      "env": {
        "API_BASE_URL": "https://example.com/api",
        "ACCESS_TOKEN": "replace-me"
      }
    }
  }
}
```

## Troubleshooting

- Authentication errors (`401` / `403`):
  - Verify `.env` contains the auth variable (`ACCESS_TOKEN`) and value format is correct.
- API unreachable / timeout:
  - Check `API_BASE_URL`, DNS/network access, and whether the API is reachable from your machine.
- Tool call fails with validation-style errors:
  - Ensure required tool inputs are provided, especially path/query parameters and request body fields.
- Server does not appear in MCP client:
  - Confirm the config path is correct and points to `dist/index.js`, then restart the client.
