# MCP Server for Picker Test API

Expose Picker Test API as MCP tools over stdio so an LLM can call the API safely.

Generated by **mcpforge** at 2026-03-06T18:22:49.748Z.

## Prerequisites

- Node.js 18+
- npm 9+

## Setup

1. Install dependencies:

```bash
npm install
```

2. Configure environment:

```bash
cp .env.example .env
```

3. Build:

```bash
npm run build
```

4. Run:

```bash
npm start
```

## Claude Desktop Configuration

## Troubleshooting

- Authentication errors (`401` / `403`):
  - This API was detected as unauthenticated; verify upstream API credentials are not required in your environment.
- API unreachable / timeout:
  - Check `API_BASE_URL`, DNS/network access, and whether the API is reachable from your machine.
- Tool call fails with validation-style errors:
  - Ensure required tool inputs are provided, especially path/query parameters and request body fields.
- Server does not appear in MCP client:
  - Confirm the config path is correct and points to `dist/index.js`, then restart the client.
